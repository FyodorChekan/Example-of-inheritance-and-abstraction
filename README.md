# Example-of-inheritance-and-abstraction
Создать решение (Solution), которое минимально содержит два проекта (Projects): исполняемый (Console Application) и библиотеку классов (Library), а опционально ещё и третий проект – модульные тесты (Tests). Разработать библиотеку классов по заданному варианту. Каждый класс необходимо размещать в отдельных двух файлах, снабжённых «говорящим именем» и специальными расширениями: *.h для заголовочных файлов (Header), содержащих API класса, и *.cpp для компилируемых (Source), содержащих реализацию класса. В запускаемом проекте требуется создать файл main.cpp, содержащий точку входа в демонстрационную программу – главную функцию (main). В рамках данной функции создать коллекцию объектов базового типа и заполнить её объектами типов-наследников. Базовый тип и типы-наследники определены в варианте задания. Проитерировать коллекцию элементов как коллекцию базового класса, выводя информацию о текущем объекте в стандартный поток вывода (std::cout)

Абстрактный базовый класс Правильная фигура, имеющий метод расчета поверхности фигуры, объема, метода сериализации в строку ToString() и статического метода чтения из стандартного потока ввода.

Реализовать класс Параллелепипед. Предусмотреть конструкторы, принимающие три вершины типа Точка, вершина типа Точка и две стороны, две вершины главной диагонали. При невозможности создать объект в случае неположительного значения стороны выбросить исключение. Реализовать методы расчета поверхности параллелепипеда, объема параллелепипеда, метода сериализации в строку ToString() и статического метода чтения из стандартного потока ввода, переопределить оператор сдвига влево.
Для класса Точка определить все необходимые для дальнейшего использования операторы (операторы сравнения, сдвига влево и вправо)
